# Goals and Background Context (v2 Roadmap)

## Goals

- Import BMad personas as content without modifying Orchestra code
- Build resource-driven infrastructure: ResourceLoader, TaskEngine, TemplateProcessor, ChecklistEngine
- Add next-gen resource types: memory, live intelligence, adaptive workflows, collaboration, predictive intelligence
- Treat personas as identity + resource stacks; expand to 16+ personas
- Preserve CLI-first local architecture, performance, and security
- Establish path to AI-driven team composition (future)

## Background Context

Orchestra’s UniversalPersona architecture proves a single execution engine with YAML-driven personas outperforms multi-service agents. Personas become identity plus a resource stack (tasks, templates, checklists, tools). BMad provides a rich content library that can be adapted to Orchestra resources. This PRD defines v2: integrate BMad as content, extend Orchestra with a general resource system, and introduce advanced resource types enabling learning, adaptation, prediction, and collaboration.
