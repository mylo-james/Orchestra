# Story 1.4: Expand CLI Persona Selection; Validate End-to-End Execution

## Status
Draft

## Story
**As a** Scrum Master,
**I want** the CLI to support selecting newly converted personas and validate end-to-end execution,
**so that** teams can use the expanded persona set via the existing command interface.

## Acceptance Criteria
1. CLI lists and selects all converted personas
2. Persona help and commands execute without code changes
3. End-to-end flow validated with at least one task/template/checklist execution
4. No regressions for existing personas (`orchestrator.yaml`, `dev.yaml`, `release.yaml`)
5. Audit logging and security monitoring capture persona executions

## Tasks / Subtasks
- [ ] Extend CLI to discover personas from resource registry (AC: 1)
- [ ] Implement selection/activation flow (AC: 1)
- [ ] Validate command execution paths (AC: 2,3)
- [ ] Regression tests for existing personas (AC: 4)
- [ ] Ensure audit/security instrumentation is intact (AC: 5)

## Dev Notes
- See `docs/current-implementation.md` for CLI interface and personas.
- Ensure no new external deps; local laptop constraints apply. See `docs/index.md`.
- Testing strategy: unit + integration + security tests. See `docs/architecture/testing-strategy.md`.

### Testing
- CLI unit tests for listing/selection
- Integration tests: activate persona and run sample task/template/checklist
- Security tests: audit logging entries present and validated
