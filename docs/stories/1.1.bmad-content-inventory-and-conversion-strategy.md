# Story 1.1: BMad Content Inventory and Conversion Strategy

## Status
Draft

## Story
**As a** Scrum Master,
**I want** a complete inventory of BMad content and a conversion strategy,
**so that** Orchestra can import personas, tasks, templates, and checklists as resources without code changes.

## Acceptance Criteria
1. Inventory of BMad personas, tasks, templates, and checklists is documented
2. Conversion rules defined to map BMad structures to Orchestra resource schemas
3. Validation approach specified for converted resources (schemas, CI checks)
4. Risks and dependencies documented (e.g., signing, provenance, versioning)
5. Output plan includes directory structure and filenames for converted resources

## Tasks / Subtasks
- [ ] Extract inventory from `.bmad-core/agents`, `tasks`, `templates`, `checklists` (AC: 1)
  - [ ] List files and categorize by type
  - [ ] Capture versions and IDs where present
- [ ] Define mapping strategy to Orchestra resource models (AC: 2)
  - [ ] Personas → YAML persona schema with identity/commands/resources
  - [ ] Tasks/Templates/Checklists → Resource definitions with schema and metadata
- [ ] Define validation and CI checks (AC: 3)
  - [ ] JSON Schemas and validators
  - [ ] CI job to validate resources on PR
- [ ] Document risks and dependencies (AC: 4)
  - [ ] Provenance/signing/trust levels implications
  - [ ] Backward-compatibility constraints
- [ ] Plan directory structure and filenames (AC: 5)
  - [ ] `orchestra/resources/{personas,tasks,templates,checklists}`
  - [ ] Versioned subfolders or metadata fields
