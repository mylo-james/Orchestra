# Story 1.2: Convert 13 BMad Personas to Orchestra YAML Personas

## Status
Spec In Progress

## Story
**As a** Scrum Master,
**I want** 13 BMad personas converted into Orchestra YAML persona format,
**so that** the UniversalAgent can load and execute commands without code changes.

## Acceptance Criteria
1. Identify 13 target BMad personas and confirm coverage
2. Produce YAML persona files conforming to Orchestra schema for each
3. Validate against persona JSON Schema in CI with helpful errors
4. Backward compatibility with existing personas (`orchestrator.yaml`, `dev.yaml`, `release.yaml`) maintained
5. Personas load under 500ms and without increased memory footprint beyond NFRs

## Tasks / Subtasks
- [ ] Select 13 BMad personas from `.bmad-core/agents` (AC: 1)
  - [ ] Capture name, role, style, commands, dependencies
- [ ] Define Orchestra persona schema mapping (AC: 2)
  - [ ] Identity, behavioral contract, commands, dependencies
- [ ] Implement schema and validator (AC: 3)
  - [ ] Add JSON Schema and CI validation step
- [ ] Verify compatibility and load performance (AC: 4,5)
  - [ ] Loader tests under `orchestra/system/loader.py`
  - [ ] Performance benchmarks for load time

## Dev Notes
- `UniversalAgent` and loader are current v0.1.0 core; no changes required. See `docs/current-implementation.md`.
- NFRs: Load time < 500ms with 16+ personas; memory footprint <= 4GB typical. See `docs/requirements.md`.
- Testing strategy enforces 90% coverage; add unit tests for schema and loader. See `docs/architecture/testing-strategy.md`.
