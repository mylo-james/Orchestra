# Story 1.9: CLI Commands for Team/Project Init and Context Switching

## Status
Spec In Progress

## Story
**As a** Scrum Master,
**I want** CLI commands for initializing team/project context and switching contexts,
**so that** users can easily manage overlays and namespaces from the CLI.

## Acceptance Criteria
1. `orchestra context init --team --project` scaffolds directories and configs
2. `orchestra context use --team --project` updates active context
3. Displays current context status and validation
4. Works across sessions; persists selection
5. End-to-end validation with loader and resources

- [ ] Implement context switching and persistence (AC: 2,4)
