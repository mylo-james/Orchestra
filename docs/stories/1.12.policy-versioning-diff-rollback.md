# Story 1.12: Policy Versioning, Diff, Rollback for Cascaded Updates

## Status
Draft

## Story
**As a** Scrum Master,
**I want** policy versioning with diff and rollback support,
**so that** cascaded updates are traceable and reversible.

## Acceptance Criteria
1. Policy versioning model with metadata (author, time, policyVersion)
2. Diff generation for before/after states
3. Rollback mechanism with audit log
4. Expiry/refresh window for gate freshness
5. CI checks to validate policy integrity

## Tasks / Subtasks
- [ ] Define versioning and metadata fields (AC: 1)
- [ ] Implement diff and storage model (AC: 2)
- [ ] Implement rollback with audit (AC: 3)
- [ ] Add optional expiry windows for gates (AC: 4)
- [ ] CI validation and examples (AC: 5)
