### Orchestrator Selection Logic

**Selection Criteria:**

- **Security Keywords**: "auth", "security", "encryption", "validation" → security-dev
- **Frontend Keywords**: "UI", "component", "React", "frontend", "user interface" → frontend-dev
- **Backend Keywords**: "API", "database", "service", "endpoint", "backend" → backend-dev
- **Mixed Requirements**: Use default dev persona or prompt user for selection

**Implementation Pattern:**

```python
def select_developer_persona(self, story_content: str, requirements: List[str]) -> str:
    """Select appropriate developer persona based on context."""
    keywords = self.extract_technical_keywords(story_content + " ".join(requirements))

    security_score = self.calculate_domain_score(keywords, SECURITY_KEYWORDS)
    frontend_score = self.calculate_domain_score(keywords, FRONTEND_KEYWORDS)
    backend_score = self.calculate_domain_score(keywords, BACKEND_KEYWORDS)

    if security_score > 0.7:
        return "security-dev"
    elif frontend_score > backend_score and frontend_score > 0.6:
        return "frontend-dev"
    elif backend_score > 0.6:
        return "backend-dev"
    else:
        return "dev"  # Default persona
```
