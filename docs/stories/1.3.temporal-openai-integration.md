## Technical Requirements Clarification

### Workflow State Schema Requirements

**Context Preservation Structure (PO/Architect Validated):**

```yaml
agent_context:
  # Core Context
  session_id: UUID # Unique conversation identifier
  conversation_history: Array[Message] # Full agent conversation thread
  current_agent: AgentType # Currently active agent
  handoff_reason: String # Why handoff occurred
  correlation_id: UUID # Cross-system tracing
  confidence_scores: Dict[String, Float] # Agent confidence in outputs
  working_memory: Dict[String, Any] # Agent-specific context data
  task_state: TaskState # Current task progress
  security_context: SecurityContext # Auth and permissions

  # Architectural Enhancements
  schema_version: String # For future migrations
  circuit_breaker_state: CBState # For degraded mode operations
  retry_metadata: RetryContext # For exponential backoff tracking
  resource_constraints: Resources # Memory/time limits per agent
```

### Activity Timeout Configuration

**Timeout Values by Operation Type (PO Specified, Architect Validated):**

- **Planning Operations:** 30s (rapid ideation, prevents UI blocking)
- **Code Generation:** 120s (complex analysis allowed, balances thoroughness)
- **PR Creation:** 60s (GitHub API reliability, accounts for variability)
- **Knowledge Operations:** 10s (fast retrieval required, optimizes real-time conversations)

**Implementation:** Adaptive timeouts that increase under load while maintaining user experience.

### Error Classification Rules

**Retry Policy (PO Requirements with Architect Enhancements):**

#### OpenAI API Errors

- **Rate Limits (429):** Retry with exponential backoff (max 3 attempts)
- **Server Errors (5xx):** Retry immediately (max 2 attempts)
- **Invalid Requests (4xx):** Fail immediately - no retry
- **Circuit Breaker:** 5 failures → 30s circuit open

#### GitHub API Errors

- **Rate Limits:** Retry with GitHub-specific backoff
- **Server Errors:** Retry with jitter (max 3 attempts)
- **Circuit Breaker:** 3 failures → 60s circuit open

#### Network/Timeout Errors

- **Timeouts:** Retry with jitter (max 3 attempts)
- **Network failures:** Exponential backoff with max 5 attempts

#### Fallback Strategies

- **OpenAI failure:** Cached response or simplified generation
- **GitHub failure:** Local git operations only
- **Pinecone failure:** In-memory knowledge store

### Workflow Architecture Requirements

[Source: architecture/high-level-architecture.md]

- Event-Driven Agent Orchestration: Temporal workflows coordinate agent handoffs through durable event sequences
- Orchestrator serves as central coordination hub managing workflow orchestration
- All external API calls (OpenAI, GitHub, Pinecone) must be wrapped in Temporal activities for durability

### Agent Handoff Patterns

[Source: architecture/components.md]

- Orchestrator manages internal handoffs to Developer and Release agents
- Developer Agent receives handoff from Orchestrator for code generation
- Release Agent receives handoff from Orchestrator for repository operations
- All handoffs must maintain session context and correlation IDs

### Coding Standards Requirements

[Source: architecture/coding-standards.md]

- Temporal Activities: All external API calls must be wrapped in Temporal activities for durability
- Error Context: All exceptions must include correlation IDs for tracing across agent handoffs
- Async Patterns: All I/O operations must be async/await for optimal Temporal integration
- Session Management: Agent conversations must maintain session context through handoff mechanisms
