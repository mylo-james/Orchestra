# Story 1.8: Namespaced Storage (Qdrant collections, PG schemas, logs)

## Status
Draft

## Story
**As a** Scrum Master,
**I want** namespaced storage across Qdrant, PostgreSQL, and logs,
**so that** data isolation by project and team is enforced and scalable.

## Acceptance Criteria
1. Namespace model defined (team, project, persona)
2. Qdrant collections and PG schemas follow namespacing rules
3. Logs include namespace fields and are queryable
4. Migration/initialization scripts provided
5. Load/perf targets validated for ≥100 projects and ≥10k docs/collection

## Tasks / Subtasks
- [ ] Define namespace schema and naming conventions (AC: 1)
- [ ] Implement Qdrant/PG isolation and initialization (AC: 2,4)
- [ ] Instrument logging with namespace fields (AC: 3)
- [ ] Performance tests and validation (AC: 5)
