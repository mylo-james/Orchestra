# Story 1.5: Persona Overlay Merge Engine (base → team → project)

## Status
Spec In Progress

## Story
**As a** Scrum Master,
**I want** a deterministic overlay merge engine for personas,
**so that** base personas can be customized by team and project without duplication.

## Acceptance Criteria
1. Merge precedence defined: base < team < project
2. JSON Schema and validation for overlays
3. Deterministic conflict resolution with audit trail
4. Hot-reload support and cache keys include (persona, team, project, version_hash)
5. CI validation with helpful errors on overlay conflicts

## Tasks / Subtasks
- [ ] Define overlay schema and precedence rules (AC: 1,2)
- [ ] Implement merge engine with conflict reporting (AC: 3)
- [ ] Hot-reload and cache key strategy (AC: 4)
- [ ] CI validations and examples (AC: 5)
