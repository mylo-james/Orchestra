# Story 1.7: Context-aware Loader API (load_persona(pid, team_id, project_id))

## Status
Done

## Story
**As a** Scrum Master,
**I want** a context-aware loader API to load personas by team and project,
**so that** the correct overlay stack is applied at runtime.

## Acceptance Criteria
1. API: `load_persona(pid, team_id, project_id)` implemented
2. Deterministic overlay application and verification
3. Logging/tracing includes team/project/tags/policyVersion
4. Graceful failure and fallback behaviors defined
5. Unit and integration tests cover key scenarios

## Tasks / Subtasks
- [ ] Implement API surface and integration points (AC: 1)
- [ ] Apply overlay merge engine results (AC: 2)
- [ ] Structured tracing/logging with correlation IDs (AC: 3)
- [ ] Error handling with retries and partial rollback (AC: 4)
- [ ] Tests for success/failure/edge cases (AC: 5)
